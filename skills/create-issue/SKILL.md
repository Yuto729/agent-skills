# Create Issue

`gh issue create` で Yuto729/daily-life リポジトリにIssueを作成する。

## ラベル判定

内容に応じて以下のラベルを自動付与する。複数該当する場合は複数付与してよい。

| ラベル   | 判定基準                                         |
| -------- | ------------------------------------------------ |
| `Bizgem` | Bizgem関連（インターン、業務、プロダクト）       |
| `松尾研` | 松尾研関連（仕事、TA、講義）                     |
| `研究`   | 研究関連（卒論、論文、実験）                     |
| `就活`   | 就活関連（選考、面接、課題、インターン応募）     |
| `Zenn`   | Zenn記事関連（執筆、ネタ、公開）                 |
| `Memo`   | メモ・記録・ログ・覚書・振り返り                 |
| `Idea`   | アイデア・ネタ・思いつき・提案・ブレスト         |
| `Life`   | 生活・プライベート（病院、手続き、買い物、予定） |

判断が難しい場合は `Memo` をデフォルトとする。

## コマンド

```bash
# 基本形
gh issue create -R Yuto729/daily-life --title "タイトル" --body "内容" --label "ラベル"

# 複数ラベル
gh issue create -R Yuto729/daily-life --title "タイトル" --body "内容" --label "Memo" --label "松尾研"
```

## ワークフロー

1. ユーザーの発言からタイトルと本文を構成する
2. ラベルを自動判定する
3. `gh issue create` でIssueを作成する
4. 作成されたIssueのURLを返す

## 注意

- タイトルは簡潔に（1行、50文字以内目安）
- 本文にはユーザーの発言の要点、背景、期限（あれば）を含める
- ユーザーが明示的にタイトルやラベルを指定した場合はそれに従う
- 期限がある場合は本文に `**期限: YYYY/MM/DD**` を含める
