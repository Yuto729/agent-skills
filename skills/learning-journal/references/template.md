# 記録テンプレート

knowledgeに記録する際は以下の構成に従う。
全セクション必須ではない。学びの内容に応じて必要なものだけ使う。

```markdown
# <タイトル>

## 背景

何をしていてこの学びが生まれたか。1-2文で簡潔に。

## 学び

本題。比較表、コード例、判断基準などを含む。

## プロジェクトでの採用 (具体的なプロジェクトに紐づく場合)

具体的なファイルパスと採用理由。

## 判断のポイント

どういう条件でどう選ぶかの基準。実務で迷ったときに参照できるようにする。
```

## 良い記録の例

- 比較表がある（パターンA vs B）
- プロジェクト内の具体的なコード参照がある（`ファイルパス:行番号`）
- 「いつ使うか」の判断基準が明確

## 避けるべき記録

- コードの貼り付けだけで考察がない
- 長すぎる（1ファイル200行以内を目安に）
